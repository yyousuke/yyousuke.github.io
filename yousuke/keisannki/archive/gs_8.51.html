AFPL Ghost Script のソースからのインストール

最終更新日 2005/06/05


ここでは、AFPL Ghost Script 8.51 のソースからのコンパイルとインストール 方法、CIDフォントを用いた日本語化について解説しています。Solaris, Vine Linux, Tru64で同じ方法が使用可能なことを確認しています。

ソースのダウンロード

Ghost ScriptのページからGhost Scriptのソースコード (ghostscript-8.51.tar.bz2)とコンパイルの際に必要な jpeg6b (jpegsrc.v6b.tar.gz), libpng (libpng-1.2.8.tar.gz), zlib (zlib-1.2.2.tar.gz) をダウンロードします。また、使用するフォントもダウンロードしておきます。
ghostscript-fonts-std-8.11.tar.gz
ghostscript-fonts-other-6.0.tar.gz


Ghost Scriptとjpeg6b, libpng, zlib のソースを展開

ダウンロードしたGhost Scriptのソースを展開します。

$ bunzip2 -c ghostscript-8.51.tar.bz2 | tar xvf -
(bunzip2 に-c オプションを付けることで標準出力に出力し、それをバッファに 入れてバッファの中身をtarで展開)

* GNU tar があれば、
$ gtar xjvf ghostscript-8.51.tar.bz2
でも展開できます。Linux の場合は tar コマンドが GNU tar になっています。

次に、jpeg6bのソースコードを展開し、Ghost Scriptのソースコードを展開したディレ クトリに移動します。この際、フォルダ名を jpeg/ としておく必要があります。

$ gunzip -c jpegsrc.v6b.tar.gz | tar xvf -
(GNU tar では $ tar xzvf jpegsrc.v6b.tar.gz)
$ mv jpeg6b Ghostscriptのディレクトリ/jpeg

libpng と zlib もGhost Script のソースコードを展開したディレクトリの下に 展開します。それぞれ、libpng/ 、zlib/ のフォルダ名に変更します。Ghost Scriptのコンパイルの為だけに使用するのであれば個別にコ ンパイルする必要はありません。

$ gunzip -c libpng-1.2.8.tar.gz | tar xvf -
$ mv libpng-1.2.8 Ghostscriptのディレクトリ/libpng
$ gunzip -c zlib-1.2.2.tar.gz | tar xvf -
$ mv zlib-1.2.2 Ghostscriptのディレクトリ/zlib


Ghost Scriptのコンパイルとインストール

Ghost Scriptのソースコードを展開したディレクトリに移動し、
$ ./configure (Makefileを作成)
$ make (コンパイル)
$ make check (コンパイルのチェック)
デフォルトでは /usr/local 以下にインストールされるので root になってから
$ make install
でインストールします。


フォントの展開

ここまでの状態では何もフォントがないのでGhost Script のフォントを展開し ます。ここではデフォルトの場合を考えますが、フォントを展開し、 /usr/local/share/ghostscript 以下に移動します。

$ gunzip -c ghostscript-fonts-std-8.11.tar.gz | tar xvf -
$ gunzip -c ghostscript-fonts-other-6.0.tar.gz | tar xvf -

fonts/ フォルダが作成されるので

$ mv fonts /usr/local/share/ghostscript

とします。

* Ghost Scriptのフォントはバージョンが違っても共通して使用できます。既に 以前のバージョンのフォントがあり、特にバージョンアップの予定がなければこの項目の作業は必要ありません。


Ghost Scriptの日本語化

Ghost Script 8.51 は日本語を表示できるように作られていますので日 本語フォントをインストールして読み込めるようにするだけで表示が可能です。
ここではCIDフォントの使用法について解説します。手元にCIDフォントを用意し ておいて下さい。CIDフォントがない場合にはフリーの和田研フォントが使用できます。
  • フォントのパスを指定
    フォントを /usr/local/share/ghostscript/Resource/CIDFont に置き、 /usr/local/share/ghostscript/8.51/lib/gs_res.ps を変更します。
    ------------------------------------------------------------------------
    /FontResourceDir (Font) .resource_dir_name
    ↓
    /FontResourceDir (/usr/local/share/ghostscript/Resource/CIDFont/) .resource_dir_name

    /GenericResourceDir () .resource_dir_name
    ↓
    /GenericResourceDir (/usr/local/share/ghostscript/Resource/) .resource_dir_name
    ------------------------------------------------------------------------

  • 代用フォントの使用
    psファイルの中では Ryumin-Light, GothicBBB-Medium という日本語フォントが 使われますが、このフォントを持っていない場合には別のフォントで代用します。 WadaMin-Regular, Wadamin-Bold で代用するには /usr/local/share/ghostscript/8.51/lib/cidfmap に
    ------------------------------------------------------------------------
    /Ryumin-Light /WadaMin-Regular ;
    /GothicBBB-Medium /WadaMin-Bold ;
    ------------------------------------------------------------------------
    のように記述します。
