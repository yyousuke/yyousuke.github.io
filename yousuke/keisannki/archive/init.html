１、設定ファイル

  ここでは、主な設定ファイルの概要を解説しています。使用するシェルやウィンドウマネージャーによっては必要ないファイルもありますので、自分のホームディレクトリに以下のファイル全てがなくても心配しないで下さい。

1.1 シェル関連

  • .login (csh, tcsh)
    login shellを csh, tcsh にした場合、端末からloginした際に読み込まれる設定ファイル。login後にシェルを起動しても読み込まれない。

  • .tcshrc (tcsh)
    tcshを起動した際に読み込まれる設定ファイル。login後にシェルを起動した際にも毎回読み込まれる。

  • .cshrc(csh,tcsh)
    cshを起動した際に毎回読み込まれる設定ファイル。tcshの場合には .tcshrc がない場合にこのファイルが読み込まれる。 .tcshrc の中で .cshrc を読み込むように設定し、 .cshrc にはcsh, tcshに共通する設定を記述することも多い。

  • .logout (csh, tcsh)
    login shellを csh, tcsh にした場合、端末からlogoutする際に実行される設定ファイル。ユーザのtmpファイルの削除や、画面のクリア、メッセージの表示などを設定することができる。

    • ＊ 以下の設定ファイルはbashを使用するユーザ向けです。bashを使用しない場合には不要ですので読み飛ばしても構いません。

  • .profile (sh, bash)
    login shellを sh, bash にした場合（rootのlogin shell を除けば、shにすることはほとんどないが）に端末からloginした際に読み込まれる設定ファイル。login後にシェルを起動しても読み込まれない。

  • .bash_profile (bash)
    login shellを bash にした場合に端末からloginした際に読み込まれる設定ファイル。login後にシェルを起動しても読み込まれない。

  • .bash_logout (bash)
    login shellを bash にした場合、端末からlogoutする際に実行される設定ファイル。

  • .bashrc (bash)
    bashを起動した際に読み込まれる設定ファイル。login後にシェルを起動した際にも毎回読み込まれる。





1.2 X関連

  • .xsession
    端末のX画面やXDMCPで飛ばされたXの画面(AstexXなどのlogin画面)からloginした場合に読み込まれる設定ファイル。

  • .xinitrc
    テキストモードから xinit や startx でXを立ち上げる場合に使用する。Xが立ち上がっている状態でのloginやXDMCPでloginする場合には不要。基本的には.xsession へのリンクとしてもよい。

  • .Xdefaults
    Xクライアントのリソース（大きさ、色、フォント、ボタンの形状等）を記述するファイル。記述がなければシステムのデフォルト値が用いられる。共通設定は/usr/lib/X11/ja_JP.ujis/app-defaults/...にある（Linux）。

  • .xmodmap
    キーコードと文字の対応を記述する。高橋研ではjp106キーボード用に設定しているので別のキーボード(Sunのワークステーション付属のものなど)を使わなければ不要。

  • .twmrc (twm)
    Window マネージャに twm を使用した場合、twm　が読み込む設定ファイル。使用するフォントやマウスのボタンと表示されるコマンドの指定、コマンドを選択した際のアクションの指定やXクライアントに枠を表示するかどうかなどを設定することができる。

  • .qvwmrc (qvwm)
    Window マネージャに qvwm を使用した場合、qvwm　が読み込む設定ファイル。

  • .gnome/ (gnome)
    Window マネージャに gnome を使用した場合、gnome　が読み込む設定ファイルを置くディレクトリ。twmで記述した内容に加え、デスクトップに表示するアイコンの設定も行える。





1.3 emacs関連

  • .emacs
    emacsの設定ファイル。設定は全てemacs lispで記述する。.emacs に全ての設定を記述するタイプと .emacs には他のファイルの読み込みの設定のみを記述して .emacs.el (Vine Linux)や .gnu-emacs(SuSE Linux)に主な設定を記述するタイプ、それらの中間で、主な設定を .emacs に記述し、他のファイルで色の設定や個人的なカスタマイズの設定を記述するタイプがある。

  • .gnu-emacs
    SuSE Linuxで使われているGNU-Emacsの設定ファイル。SuSE Linuxでは、.emacsに主な設定を記述せず、このファイルに設定を記述して .emacs から読み込むように設定されている。

  • .gnu-emacs-custom
    SuSE Linuxで使われているGNU-Emacsのカスタム設定ファイル。問題が起こった場合に解決が遅れるもととなるので、個人的な設定はここに記述した方がよい。

  • .emacs.hilit
    emacsのカラー設定ファイル(一般的でない可能性あり)。高橋研のTru64では .emacsから読み込むように設定しているが、色の設定を行わない場合や.emacsに色の設定も含めた場合にはこのファイルがなくてもよい。

  • .emacs.el
    Vine Linux で使われているemacsの設定ファイル。Vine Linuxでは.emacs の代わりにこのファイルにシステムの主な設定が記述される。

  • .emacs.my.el
    Vine Linux で使われているemacsの個人設定ファイル。.emacs, .emacs.el は自動的に更新されるため、ここに個人的な設定を記述する。

  • .emacs-color.el
    emacsのカラー設定ファイル。Vine Linuxで使用されていますが、個人的に .emacs-color.el, .emacs-faces.el, .emacs-fontset.el を他のシステムでも使用できるように設定しているので、設定ファイルを見る際には注意して下さい。

  • .emacs-faces.el
    emacsのフォントセットの設定とフォントの装飾、背景色の設定などを記述する(Vine Linux)。

  • .emacs-fontset.el
    emacsのフォント指定を行うファイル(Vine Linux)。

  • .canna (canna)
    cannaの設定ファイル。cannaを使用する場合、ユーザ辞書の指定はここで行います。

  • .eggrc
    tamagoの設定ファイル。tamagoでwnnを使用する場合、辞書の設定はここで行います。

  • .xemacs.el
  • .xemacs/init.el
    xemacsの設定ファイル。Linuxの場合、.emacsでこれらのファイルか.emacs.elを読み込むか切替えるようにしている。.emacs に主な設定を記述するシステムでもこれらのファイルを使用することが多い。





1.4 vi関連

  • .exrc
    viを起動する際に毎回読み込まれる設定ファイル。viの最終行モードで入力するコマンドを記述することができます。最終行モードの最初の:は必要ないので注意してください。
    次に設定例を載せておきます。 
    ---------------------------------------------
" 括弧の対応を表示する"
set showmatch
set ignorecase
" 自動インデント機能を有効にする"
set autoindent
" カーソルがある行・列を表示する(vimのみ)"
set ruler
" 編集モードの時に画面の下に表示が出るようにする。"
set showmode
" ソースコードの文法を解釈して表示するかどうか設定する(対応していないものも多いので注意)。"
" syntax on"
syntax off
" バックアップファイルを作らない "
set nobackup
---------------------------------------------





1.5 Mail関連

  • .forward
    メール転送の設定ファイル。このファイルの中にアドレスを記述するとそのアドレスにメールが転送される。何箇所にも転送したい場合には2行目、3行目、・・・にもアドレスを記述する（確か転送先の数に制限があったと思うのですが忘れました）。自分の所にもメールを残したい場合には自分のアドレスを書く必要があるので注意して下さい。
    くれぐれも.forward の設定がループにならないように（アドレスAからB、BからC、CからAへ転送など）して下さい。
    user1@host1.hoge.ac.jp にメールを残し、user2@host2.hogehoge.ne.jpに転送したい場合には次のようにします。 
    ---------------------------------------------
user1@host1.hoge.ac.jp
user2@host2.hogehoge.ne.jp
---------------------------------------------

  • .mew.el
    emacsでMewを使用するときの設定ファイル。emacsでMailを読むことのないユーザには不要。





1.6 その他の設定ファイル

  • .rhosts
    この中にユーザ名とホスト名を記述すると、そのホストの指定したユーザは rlogin, rshなどをパスワードなしで実行できるようになる。セキュリティ上問題があるので置かない方がよい。多くのマシンでrlogind, rshd は起動しないので、このファイルを置いても何も起こらない。

  • .shosts
    上と同じだが、sshを使った場合にパスワードなしでloginできる。デフォルトで設定されているはずだが、sshdの設定ファイルで.shostsを無視するように設定した方がよい。

  • .dtprofile
    Tru64やSolarisでCDE（共通デスクトップ環境）を使用する場合に読み込まれる設定ファイル。環境変数の設定など。

  • .gmtdefaults
    GMTのプログラムを実行する際に読み込まれる設定ファイル。1) システムのデフォルト値、2) ホームディレクトリの .gmtdefaults、3) current ディレクトリの .gmtdefaults の順で読み込まれる。それまでの設定を上書きするので、後に読み込まれるファイルの設定が優先される。gmtset コマンドを実行すると自動的に作成されるので注意が必要。

  • .xscreensaver
    スクリーンセーバーの設定ファイル。

  • .dir_colors
    lsで表示されるカラーを設定するファイル。Linuxでは /etc/DIR_COLORS にデフォルトの設定ファイルがある。環境変数LS_COLORSの方が優先される。





1.7 設定ファイルではない.ファイル

  • .history
    コマンド履歴を保存するファイル

  • .bash_history
    bashの場合にコマンド履歴を保存するファイル

  • .gp_history
    gnuplot のコマンド履歴を保存するファイル

  • .gmtcommand
    GMTのコマンドを実行した際にオプションを保存するファイル

  • .emacs.d/
    emacsが自動的にセーブしたファイルを保存するディレクトリ

  • .dt/
    xsessionのログファイルを保存するディレクトリ

  • .ssh/
    sshでloginする際のホストキーやssh-keygenで生成する秘密鍵・公開鍵を保存する。このディレクトリは他人から見られないようにパーミッションを700にする必要がある(なっていなければ鍵認証でloginできない)。
